百约 · BaiYue v3.0 完整教程 — Python + NapCatQQ + DeepSeek

Python 3.12 NapCatQQ DeepSeek API OneBot v11 WebSocket WebUI 配置面板

你在QQ发消息
  ↓
NapCatQQ 拦截消息 → 通过 WebSocket 转发给 Python
  ↓
Python 把消息发给 DeepSeek（AI大脑）
  ↓
DeepSeek 生成回复 → Python 发回 NapCat → QQ
  ↓

NapCatQQ翻译官。把QQ消息翻译成Python能读的JSON格式
Python程序中间人。收消息、调AI、发回复
DeepSeek API大脑。真正"说话"的那个

📦 第一步：安装 Python

  1. 打开 python.org，下载 Python 3.12.x
  2. 安装时勾选 "Add Python to PATH"（非常重要！）
  3. 验证：python --version 应显示 Python 3.12.x
  4. 装依赖：python -m pip install websockets requests edge-tts
⚠️ 不要从微软商店装 Python！商店版是个空壳，python --version 没输出、pip 装不了包。从 python.org 下载正式版。

🔌 第二步：安装 NapCatQQ

  1. 下载：github.com/NapNeko/NapCatQQ/releases
  2. 下载 NapCat.Shell.zip，解压到某目录（如 D:\NapCat）
  3. 双击 NapCatWinBootMain.exe
  4. 弹出二维码 → 用手机QQ（机器人号）扫码登录
  5. 验证：浏览器打开 http://127.0.0.1:3000/get_status，看到 "online":true

🔑 第三步：获取 DeepSeek API Key

  1. 打开 platform.deepseek.com
  2. 注册 → API Keys → 创建 Key → 复制保存
  3. 充值 10 块钱，够用几个月

💰 费用参考：聊100条约0.3元，聊一个月不到20块。语音用微软免费TTS，不花钱。

🔗 第五步：配置 NapCat（最容易出错的步骤！）

NapCat 需要配置两个东西：HTTP 服务（用来发消息）+ WebSocket 客户端（用来收消息）。少了任何一个百约都不工作。

5.1 进入 NapCat WebUI

  1. 在 NapCat 启动窗口里找到 WebUi Token: xxxxxxxx（一长串字符）
  2. 浏览器打开 http://127.0.0.1:6099/webui?token=你的Token
  3. 如果 Token 过期了，重启 NapCat 会生成新的

5.2 配置 HTTP 服务（发消息用的）

  1. 左侧菜单 → 「HTTP 服务」
  2. ✅ 开启 HTTP 服务（默认端口 3000，不用改）
  3. ❌ 关闭 Token 验证（重要！不关的话百约请求会被拒绝）
  4. 点击保存
⚠️ 这个 Token 不是 DeepSeek 的 Key！是 NapCat 自己的 HTTP 访问令牌。百约目前不支持带 Token 的连接，所以必须关掉。

5.3 配置 WebSocket 客户端（收消息用的）

  1. 左侧菜单 → 「网络配置」→「WebSocket 客户端」
  2. 点击「添加」：
字段填什么说明
名称百约随便填，自己能认出来就行
URLws://127.0.0.1:8001必须是 ws:// 不是 http://
重连开启断了自动重连
  1. 保存
⚠️ 常见错误：
· 把 ws:// 写成了 http:// → WebSocket 必须用 ws 协议
· 忘了关 HTTP Token 验证 → 百约发消息被拒
· 只配了 HTTP 服务忘了配 WebSocket → 收不到消息
· 只配了 WebSocket 忘了开 HTTP 服务 → 发不出消息
· WebSocket 地址里多打了空格 → 连接失败

5.4 验证配置

两个都配好后：

  • 浏览器打开 http://127.0.0.1:3000/get_status → 看到 "online":true
  • 启动 bot.py 后看到 [连接] NapCat 已接入

python bot.py

👥 群聊怎么用

  1. 把机器人拉进群
  2. 在群里 @机器人 或喊「百约」「小约」
  3. 只有 @了或喊了触发词才会回复，平时不会打扰

🎤 语音消息怎么用

百约平时不会主动发语音。只有你对她说以下关键词时才会：

"说句话" · "发语音" · "叫一声" · "想听你的声音" · "来个语音" · "说句话听听" · "讲句话"

WebUI 语音设置页有 29 种音色可选，点 ▶ 可以试听。

🩺 常见问题排查（真实踩坑记录）

这些都是百裏和社区用户实际踩过的坑。按症状查找：

安装类问题

🔴 python 不是内部命令
原因：Python 没装好，或没勾选 "Add to PATH"
✅ 解决：重装 Python，安装界面一定勾选底部 "Add Python to PATH"。或每次用 python -m pip 代替 pip
🔴 微软商店版 Python 不工作
现象：python --version 没输出、pip 装不了包
✅ 解决：卸载商店版，去 python.org 下载正式版。商店版是个空壳！

连接类问题

🔴 连不上 NapCat（端口 3000 打不开）
原因：NapCat 没启动 / QQ 没登录 / HTTP 服务没开
✅ 解决：(1)确认 NapCatWinBootMain.exe 正在运行 (2)确认已扫码登录QQ (3)去 NapCat WebUI 确认 HTTP 服务已开启
🔴 NapCat 有 Token 验证导致请求被拒
现象：bot.py 日志显示 401/403 错误
✅ 解决：NapCat WebUI → HTTP 服务 → 关闭 Token 验证
🔴 机器人不回消息
几种可能：WebSocket 没配好 / 群聊没@机器人 / BOT_QQ 没填
✅ 解决：
(1) NapCat WebUI → 网络配置 → WebSocket客户端 → 确认有 ws://127.0.0.1:8001
(2) 群聊必须 @机器人 或喊触发词
(3) WebUI 账号配置页确认 BOT_QQ 填了机器人号
🔴 群聊 @了机器人但没反应
QQ 群聊 @人 实际是 CQ 码 [CQ:at,qq=xxx]，不是文字
✅ 解决：v2.1+ 已修复。确认 WebUI 里 BOT_QQ 填了机器人的 QQ 号，bot.py 会识别 CQ 码

API 类问题

🔴 百约回复"信号不太好，等会儿再说"
原因：DeepSeek API Key 填错了 / 过期了 / 余额用完了
✅ 解决：
(1) 打开 platform.deepseek.com
(2) 检查 API Keys 是否正确
(3) 检查余额，不足就充 10 块
(4) 把正确 Key 填进 WebUI → 配置页 → 保存

运行类问题

🔴 改了代码但没生效
Python 的 .pyc 缓存文件用的是旧版本代码
✅ 解决：删掉项目目录下的 __pycache__ 文件夹 → 重启程序 → 浏览器 Ctrl+Shift+R 强制刷新
🔴 端口 8001 被占用
旧的百约没关 / 其他程序占着端口
✅ 解决：任务管理器杀掉所有 python.exe。v2.0+ 会自动探测 8001-8005，被占就试下一个
🔴 bot.py 显示"等待 NapCat 启动"然后超时
NapCat 没运行 / QQ 没登录 / HTTP 服务没配好
✅ 解决：确认 (1)NapCat 正在运行 (2)QQ 已扫码登录 (3)HTTP 服务端口3000已开启
🔴 百约总是发语音
v2.x 人格提示词告诉了 AI 能发语音，AI 自己决定发语音太频繁
✅ 解决：v3.0 已修复！AI 平时完全不知道能发语音，只有你说"说句话/发语音/叫一声"才触发。更新到最新版 bot.py + webui.py 即可
🔴 配置文件写坏了，WebUI 打不开
手动改 config.json 时格式写错
✅ 解决：v3.0 每次保存自动备份 config.json.bak。把 config.json.bak 重命名为 config.json 即可恢复

其他

🔴 百约的记忆乱了 / 想从头开始
✅ 解决：对机器人发 "/清空" 或 "/reset" 或 "/忘记"。或在 WebUI → 记忆管理 → 找到那个用户 → 清空记忆
