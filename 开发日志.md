## 2026-06-25 — 百约诞生

**做的事：**
- 搭建了 QQ AI 机器人「百约」的完整架构：NapCatQQ (OneBot v11) → Python WebSocket → DeepSeek API → QQ 回复
- 写了第一版 bot.py，约 300 行
- 双人格系统：对百裏是 AI 女友模式，对其他人是酷酷 AI 助手

**踩的坑：**
- 微软商店版 Python 是个空壳，`python --version` 没输出、pip 装不了包 → 从 python.org 装了 Python 3.12
- `pip install` 在 CMD 里不识别 → 用 `python -m pip install` 替代
- NapCat HTTP 服务默认不开 → 进 WebUI 手动打开 3000 端口
- AstrBot 占着 8001 端口，百约起不来 → 后来 v2.0 加了自动端口探测
- NapCat 反向 WebSocket 不会自动重连 → 需要去 WebUI 手动触发

**关键决策：**
- 选 DeepSeek 不选 Ollama 本地模型：API 便宜（一月不到 20 块），效果比 3B 小模型好太多
- 选 NapCat 不选 AstrBot 框架：自己写几百行代码比用别人框架更可控
- 双人格分人：`OWNER_QQ` 决定谁是男朋友，不是硬编码写死

## 2026-07-04 — 配置变量化 + 群聊@修复 + 旧格式兼容

**配置变量化：**
- 新增 `OWNER_NAME` 配置变量：主人称呼不再是硬编码"百裏"，改配置即可
- 新增 `BOT_NAME` 配置变量：机器人名字不再是硬编码"百约"，改配置即可
- 新增 `BOT_QQ` 配置变量：机器人 QQ 号，用于识别群聊 @提及
- `BOT_NAME` 影响范围：提示词人格、群聊触发词、所有日志打印、启动横幅、退出语
- 非主人提示词加上 f-string 前缀，让 `{OWNER_NAME}` 可以正常插值
- 现在别人部署只改文件开头 4 个变量：`OWNER_QQ` / `OWNER_NAME` / `BOT_NAME` / `BOT_QQ`

**群聊 @提及修复：**
- 问题：QQ 群聊 @机器人 会变成 CQ 码 `[CQ:at,qq=xxx]`，不是文字"@百约"，触发词匹配不上
- `is_calling_me` 新增检测 `[CQ:at,qq=机器人QQ]` 的逻辑
- `clean_message` 新增清理 `[CQ:at,qq=...]` CQ 码，避免 AI 看到乱码

**群聊女友语气调整：**
- 原来"私聊可以甜一点，群聊收敛" → "私聊可以放开甜，群聊保持甜蜜但带点害羞"
- 效果：群里 @她也会甜，只是带着"当着大家面呢"的自然害羞感

**踩的坑：**
- 旧格式记忆文件（v1.0 纯数组）导致 `AttributeError: 'list' object has no attribute 'get'`
  → `load_memory` 加了 `isinstance(data, list)` 检测，自动包装成新格式 `{recent: [...], summary: ""}`
- `.pyc` 缓存导致改了代码不生效 → 手动删 `__pycache__` 解决
- WebSocket 异常导致连接断开但没日志 → `handle_ws` 加了 catch-all 异常捕获和 `flush=True`

**改了什么：**
- 之前提示词里"百裏""百约"写死在代码里，别人部署要翻到提示词深处去改
- 之前群聊 @机器人 靠文字匹配，但 QQ 实际发的是 CQ 码，群聊一直没回复
- 现在只改文件开头 `OWNER_QQ` / `OWNER_NAME` / `BOT_NAME` / `BOT_QQ` 四个变量即可

## 2026-07-08 — WebUI 配置面板 + 语音 + 戳一戳 + 预设人格

### 🎨 WebUI 网页配置面板（webui.py）
- 粉白配色 + 左侧导航布局，浏览器打开 `http://127.0.0.1:8080`
- **配置页**：填 API Key / QQ 号 / 称呼，不用找行号改代码
- **人格页**：预设选择器 + 自由编辑文本框，支持变量 `{BOT_NAME}` `{owner_name}`
- **语音页**：试听按钮预览音色 → 点击选中 → 保存
- 配置自动存到 `config.json`，bot.py 启动时自动加载覆盖默认值
- `config.json` 加入 .gitignore，不会泄露隐私

### 🎙 语音消息（edge-tts，免费）
- 安装 `edge-tts`（微软免费 TTS），不消耗 token
- AI 在回复开头加 `[语音]` 标记时才发语音，不会每条都说
- 触发场景：暧昧时刻、用户说"说句话""发语音"、撒娇时
- 语音生成在后台线程，不阻塞文字回复
- 7 种中文音色可选：晓晓/晓依/晓雨/晓北/晓妮/云希/云扬

### 👆 戳一戳回复
- 检测 OneBot v11 的 poke 事件，目标为机器人时自动回复
- 主人戳 → 8 条甜宠回复词库随机选
- 别人戳 → 6 条冷淡拒绝词库随机选
- 私聊群聊均支持

## 技术要点备忘

**架构：** `QQ → NapCat(WS 8001) → bot.py → DeepSeek API → NapCat(HTTP 3000) → QQ`

**关键配置（别人部署时需要改的）：**
- 用浏览器打开 `http://127.0.0.1:8080` 在网页里改，不用翻代码！
- 或者手动编辑 `config.json`，启动后自动生效
- 代码里 `bot.py` 的默认值会被 `config.json` 覆盖

**NapCat 配置（别人部署时需要做的）：**
1. HTTP 服务器：开 3000 端口，关 Token 验证
2. WebSocket 客户端：加 `ws://127.0.0.1:8001`
